# Эскиз

Эмулятор представляет собой консольное приложение. При запуске пользователь указывает коэффициент ускорения работы станка, скорость и количество строк-событий, способ ротации и кодировку файлов, которые также могут быть заданы по умолчанию. Программа работает в бесконечном цикле пока не заполнит нужное количество строк или до прерывания пользователем с помощью комбинации Ctrl+C. Она генерирует события двух типов: штатные и ошибки в разных параллельных потоках: работают параллельно, но разделяют общие файлы через Mutex. Программа будет создавать файлы с заложенными заранее наименованиями (error.txt, operate.txt) в выделенной папке, путь к которой будет указан по завершении программы. При достижении размера любого из файлов в 5 мб происходит ротация.

# Словари и пояснения
 
Словарь аргументов командной строки:
1. speed-op - положительное целое число - скорость генерации логов
2. speed-err - положительное целое число - скорость генерации ошибок
3. accel - положительное целое число - ускорение времени
4. max - положительное целое число - максимальное количество строк (логов)
5. strategy - 1/2 - стратегия сохранения файла
6. encoding - cp1251/ascii/utf8 - кодировка логов
7. time-mask - YYYY-MM-DD HH:MIN:SS.MMM / HH:MIN:SS.MMM / MM/DD/YYYY HH:MIN

Блокировка доступа к файлу в ходе записи - перед каждой записью поток убеждается, что только он сейчас хочет открыть файл, чтобы избежать одновременной записи в один файл из разных потоков.

Словарь тегов в строке:

1. {HEX[N]} - N шестнадцатеричных цифр
2. {INT} - целое число
3. {STATE_OLD} - целое число - старый статус машины
4. {STATE_NEW} - целое число - новый статус машины

Процесс формирования строки: шаблоны строк хранятся в папке source, на месте тегов программа подставляет псевдослучайные числа, далее перед строкой подставляется временная метка и в таком виде готовая строка передается на запись в соответствующий поток.

Все блок-схемы проекта выполнены в программе PlantUML.
